<a id="1"></a>第 1 章：产品定义与范围

1.1 产品愿景

为技术用户提供一款高度克制、聚焦对话效率的 AI 聊天客户端。支持多模型、多会话、流式响应与完全本地化存储，让用户掌控数据与交互节奏。

1.2 应用定义

一款个人 AI 聊天助手，用户可自行配置 API Key（或通过服务端代理），支持流式对话、Markdown 渲染、多会话管理、模型参数调节、系统提示词定制。数据完全存储在本地，不上传至任何服务器。

1.3 目标用户画像

· 主要用户：开发者、技术写作人员、AI 重度使用者
· 次要用户：期望自定义模型参数、API 端点与系统提示词的高级用户
· 潜在用户：关注隐私，希望数据仅存储在本地的普通用户

1.4 核心价值主张

· 纯粹对话：界面去装饰化，最大化内容可视区域
· 效率工具：模型快速切换、模式分层（深度思考/联网搜索）、流式响应可中断
· 隐私可控：本地加密存储、云端同步开关、内容举报机制
· 专业配置：Temperature 细粒度调节、自定义 API 端点、系统提示词模板

1.5 核心目标

· 实现高质量、高性能的流式 AI 聊天体验
· 展示现代 Android 开发最佳实践：Compose, MVVM, Hilt, Room, Kotlin Coroutines/Flow
· 保障用户数据安全，符合 Google Play 对生成式 AI 应用的合规要求

1.6 目标学员画像（教学版本）

· 已熟悉 Kotlin 基础、协程基本用法
· 了解 Compose 的基本组件（Column, Row, LazyColumn 等）和状态管理
· 了解 Room、Retrofit 的基本使用
· 希望完成一个真实、可上架的 AI 应用

<a id="2"></a>第 2 章：功能需求定义

2.1 MVP 功能（必做）

模块 功能描述 优先级
聊天对话 发送文本消息；接收流式 AI 回复（打字机效果）；停止生成；重新生成；长按复制/删除消息 P0
会话管理 新建会话；切换历史会话；重命名；删除；搜索 P0
Markdown 渲染 支持标题、列表、代码块高亮、表格、链接、粗斜体等，流式过程中增量渲染无闪烁 P0
模型选择 底部快捷切换已配置模型；底部弹窗列表选择；自定义模型配置入口 P0
模式切换 深度思考/联网搜索/快速模式切换 P1
系统提示词 自定义 System Prompt；提供预设模板（翻译助手、代码专家、写作助手） P1
外观设置 深色/浅色主题；中/英语言切换；字号调节（小/中/大三档，实时预览） P1
API Key 安全 加密本地存储 (EncryptedSharedPreferences)；安全提示弹窗 P0

空状态引导 横向提示词卡片，点击自动填入并发送 P1
内容合规 内容举报入口、隐私政策链接 P0（上架必须）

2.2 增强功能（可选）

功能 描述 优先级
语音输入 使用 SpeechRecognizer 进行语音转文字 P2
TTS 朗读 使用 TextToSpeech 朗读 AI 回复 P2

2.3 非功能需求

类别 要求

可靠性 网络异常、API 限流等有友好提示和重试机制；“停止生成”能真正中断网络连接
可测试性 Repository 和 ViewModel 具备单元测试条件；提供关键流程测试用例
可维护性 分层清晰，模块间低耦合；核心聊天模块独立，便于更换 AI 后端
合规性 符合 Google Play 生成式 AI 应用政策：内容举报入口、隐私政策、禁止生成受限内容声明
多语言 支持中文、英文切换

<a id="3"></a>第 3 章：用户界面设计规范

3.1 设计语言与原则

核心设计理念：源自 DeepSeek “极简、克制、高效”的设计哲学

· 去装饰化：减少视觉噪音，用留白代替分割线，避免过度使用阴影、渐变
· 层级分明：通过背景色区分表面容器（Surface/Surface Variant）与底层，不依赖阴影
· 聚焦对话：聊天界面为绝对核心，其他功能收起到侧边栏/设置
· 尺寸自然化：按钮、卡片等组件尺寸与文字大小相匹配，不刻意放大填充空间
· 效率优先：常用操作（模型切换、模式开关）放置在一级触达区域，减少跳转
· 隐私可控：数据本地存储，设置中提供清晰的隐私管理入口
· 状态可见：流式回复带有光标动画，生成中可随时停止，消息可长按操作

3.2 信息架构

App

3.3 页面详细设计

3.3.1 聊天主界面

布局：全屏纵向排列，透明状态栏 + 顶栏 + 对话区 + 输入区 + 工具栏。

区域 内容 交互
顶栏 左侧汉堡图标，中间动态标题（初始“新对话”，发送消息后截取首条用户输入前10字概括），右侧占位 点击汉堡图标打开侧边栏
对话区 空状态：居中引导语“✨ 开启一段新对话” + 横向提示词卡片（从左到右排列，可换行）；有消息时：用户气泡右对齐（紫色圆角），AI 气泡左对齐（浅灰圆角），流式输出时末尾有闪烁光标“▌” 长按任意气泡弹出操作菜单（复制/重新生成/删除）；点击提示词卡片自动填入并发送
输入区 圆角输入框 + 发送按钮（40×40dp 圆形） 输入框为空时按钮灰色不可用，有内容时变紫色可用；发送后按钮变为红色停止按钮（■），点击可中断生成；生成完毕恢复为紫色发送按钮；支持回车发送
底部工具栏 三个横向等宽按钮：深度思考（默认选中）、联网搜索、模型选择 点击深度思考/联网搜索切换选中态；点击模型选择从底部弹出模型列表弹窗

空状态提示词卡片规范：

· 布局：横向排列（flex-wrap: wrap），左对齐，间距 8dp
· 卡片样式：背景 --surface，圆角 8dp，边框 1px solid var(--border)，内边距 8dp 12dp，文字 14sp
· 无最小宽度/高度，宽度由内容撑开，确保“圆圈包住字”的简洁感
· 示例卡片：💡 优化Python性能、🧠 Transformer架构、✍️ 夏日诗歌

消息气泡规范：

· 用户气泡：max-width: 82%，背景主色 #6750A4，白色文字，右下角 6dp 圆角
· AI 气泡：max-width: 82%，背景 #F2F2F5（浅）/ #2A2A2A（深），左下角 6dp 圆角
· 流式光标：使用 AnnotatedString 末尾追加闪烁 ▌ 字符

3.3.2 模型选择弹窗（底部滑出）

· 标题“选择模型”，右上角关闭按钮
· 列表项：已配置的模型列表（如 DeepSeek-V3、DeepSeek-Coder、通用对话模型），当前选中项左侧有勾选标记
· 选择后按钮文字更新为模型名称，弹窗关闭
· 尺寸限制：最大高度 40% 屏幕高度，超出滚动；移动端全宽，桌面端最大宽度 480dp

3.3.3 侧边栏

· 顶部：标题“历史对话”
· 搜索框：圆角输入框，带搜索图标
· 对话列表：每个条目单行截断（ellipsis=end），当前活跃对话左侧有 3dp 紫色竖线指示条，点击条目加载对话并关闭侧边栏
· 底部栏：左侧圆形头像 + “个人中心”文字，右侧更多按钮（⋮），点击整个底部区域进入账号管理页（非直接进入设置）

侧边栏宽度：clamp(260dp, 75vw, 320dp)，遮罩层半透明黑色。

3.3.4 设置页

分组卡片式布局，无阴影，仅通过背景色和边框区分。

· 账号：账号管理（带 › 箭头）
· 通用设置：
  · 深色模式：开关组件
  · 语言：下拉选择（中文/English）
  · 字号：下拉选择（小/中/大，对应 14/16/18sp 基准），下方实时预览文字“预览文字效果 Preview”随字号变化
· 模型配置：
  · 默认模型：下拉选择（DeepSeek-V3、DeepSeek-Coder、通用对话模型等）
  · Temperature：滑块（0.0 - 2.0，步长 0.2），右侧实时显示当前数值
  · 自定义大模型配置：入口（›）→ 跳转配置页
· 隐私：
  · 清除所有对话（红色文字，点击二次确认）
  · 内容举报（›）→ 跳转举报表单或邮件

3.3.5 账号管理页

· 顶部返回按钮 + 标题“账号管理”
· 居中大头像（64dp圆形）+ 邮箱地址
· 列表项：修改头像、修改密码、绑定手机
· 设置入口：点击跳转至设置页
· 退出登录：红色文字

3.3.6 自定义模型配置页

· 顶部返回按钮 + 标题“自定义模型配置”
· 输入字段：API 地址、模型名称、API Key（密文输入）
· 底部保存按钮（紫色全宽）
· 保存后该模型可出现在模型选择弹窗中

3.4 关键交互流程

3.4.1 发送消息与流式回复

1. 用户在输入框输入文字 → 按钮变紫色可用
2. 点击发送（或回车）→ 用户气泡出现，AI 气泡出现并开始流式追加文字，光标闪烁，按钮变为红色停止按钮（■）
3. 生成完毕 → 光标消失，按钮恢复紫色发送状态
4. 点击停止按钮 → 流式中断，AI 回复保留已生成部分，按钮恢复

3.4.2 侧边栏与导航

1. 点击汉堡菜单 → 聊天页右移，侧边栏从左滑入，遮罩出现
2. 选择历史对话 → 加载该对话内容，侧边栏关闭，标题更新
3. 点击侧边栏底部 → 进入账号管理页，侧边栏自动关闭

3.4.3 设置与子页面

1. 从账号页点击“设置”或通过侧边栏底部入口（进入账号页后跳转）访问设置
2. 设置中子页面（账号管理、自定义模型配置）均以全屏滑入方式呈现，左上角返回

3.4.4 模型快速切换

1. 点击底部“模型选择”按钮 → 底部弹窗上滑显示模型列表
2. 选中模型 → 按钮文本更新为模型名称，弹窗关闭，下次对话使用新模型

3.4.5 消息长按操作

1. 长按任意消息气泡 → 弹出上下文菜单
2. 菜单选项：复制（复制消息文本）、重新生成（移除该条 AI 回复并重新请求）、删除（移除该条消息）

<a id="4"></a>第 4 章：系统架构设计

4.1 架构模式

采用 MVVM + Repository 模式，遵循单向数据流：

```
UI Layer (Compose Screens)
       ↕ StateFlow (只读状态)
ViewModel Layer (业务逻辑，状态持有)
       ↕ Repository (数据操作)
Repository Layer (数据协调)
   /            \
Local Sources    Remote Sources
(Room, DataStore) (Retrofit, SSE)
```

// 方式一：使用可刷新的 OkHttpClient（推荐）
@Singleton
@Provides
fun provideOkHttpClient(
    settingsRepository: SettingsRepository,
    @ApplicationScope coroutineScope: CoroutineScope
): OkHttpClient {
    var currentBaseUrl = runBlocking { settingsRepository.getBaseUrl() }
    
    // 监听设置变化，动态更新 client（需要配合 EventListener 或拦截器）
    val interceptor = Interceptor { chain ->
        val originalRequest = chain.request()
        // 每次请求前获取最新的 baseUrl（非阻塞）
        val baseUrl = runBlocking { settingsRepository.getBaseUrl() }  // 注意：仅用于示例，实际应避免 runBlocking，见下方替代方案
        // 替代方案：维护一个可变 StateFlow，在协程中收集更新，但 Interceptor 中无法挂起，
        // 因此推荐使用方法二（重建 client）或使用 dynamicUrl 参数。
        chain.proceed(originalRequest)
    }
    // 实际生产建议：在设置保存时重建 OkHttpClient 并重新注入 Retrofit。
    // 简化方案：在 Interceptor 中不动态切换 host，而是让用户在设置更改后提示重启应用。
    return OkHttpClient.Builder()
        .addInterceptor(interceptor)
        .build()
}



7.3 SSE 事件流解析器（增强版）
kotlin
class EventSourceParser {
    fun parse(source: BufferedSource): Flow<String> = callbackFlow {
        try {
            while (!source.exhausted()) {
                val line = source.readUtf8Line() ?: continue
                // 忽略注释行（以冒号开头）
                if (line.startsWith(":")) continue
                // 处理 data: 字段
                if (line.startsWith("data: ")) {
                    val data = line.removePrefix("data: ").trim()
                    if (data == "[DONE]") {
                        channel.close()
                        break
                    }
                    // 过滤空 data（某些实现可能发送空 data 行）
                    if (data.isNotEmpty()) {
                        trySend(data)
                    }
                }
                // 支持 \r 结尾的兼容（readUtf8Line 已处理，无需额外代码）
            }
        } catch (e: Exception) {
            if (e is CancellationException) throw e
            close(e)
        } finally {
            source.close()
        }
        awaitClose { source.close() }
    }
}

7.4 AiRepository 实现要点

```kotlin
class AiRepository @Inject constructor(
    private val apiService: OpenAiApiService,
    private val settingsRepository: SettingsRepository,
    private val json: Json
) {
    fun streamChat(
        messages: List<Message>,
        systemPrompt: String
    ): Flow<String> = flow {
        val apiKey = settingsRepository.getApiKey()
        val model = settingsRepository.getDefaultModel()
        val temperature = settingsRepository.getTemperature()

        val requestMessages = mutableListOf<RequestMessage>()
        if (systemPrompt.isNotBlank()) {
            requestMessages.add(RequestMessage("system", systemPrompt))
        }
        requestMessages.addAll(messages.map {
            RequestMessage(it.role, it.content)
        })

        val request = ChatCompletionRequest(
            model = model,
            messages = requestMessages,
            temperature = temperature,
            stream = true
        )

        val response = apiService.streamChatCompletion(
            auth = "Bearer $apiKey",
            body = request
        )

        val parser = EventSourceParser()
        parser.parse(response.source()).collect { data ->
            val chunk = json.decodeFromString<ChatCompletionChunk>(data)
            val content = chunk.choices.firstOrNull()?.delta?.content ?: ""
            if (content.isNotEmpty()) emit(content)
        }
    }.flowOn(Dispatchers.IO)
}
```
